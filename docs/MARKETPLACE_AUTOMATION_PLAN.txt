================================================================================
CRAYONSPARKS — MULTI-MARKETPLACE PUBLISHING AUTOMATION PLAN
Etsy + Pinterest + Gumroad — for both Coloring Books and Story Books
================================================================================

OVERVIEW
--------
Build a publishing automation layer that takes a generated book (coloring or
story) and pushes platform-specific listings/posts to Etsy, Pinterest, and
Gumroad. The system reuses existing artifacts (covers, mockups, PDFs, KDP
metadata) where possible, regenerates platform-shaped assets where necessary,
and provides a single UI surface to publish to one or more marketplaces with
idempotent partial-recovery semantics.


REQUIREMENTS
------------
- Support both COLORING BOOKS and STORY BOOKS (flag the latter if not built)
- Three marketplaces: Etsy, Pinterest, Gumroad
- Reuse existing Gemini-generated artifacts (covers, mockups, interior, PDFs,
  ZIP) wherever the platform allows
- Auth-token storage encrypted at rest and rotatable
- Background-job execution with partial-failure recovery (no UI blocking)
- Per-platform compliance (Etsy AI disclosure mandatory)
- Phased rollout: Gumroad → Pinterest → Etsy (low → high friction)


ASSUMPTIONS / OPEN QUESTIONS
----------------------------
1. Story-book pipeline: confirm whether /api/generate already produces story-
   book artifacts (illustrated pages + narrative text) or only coloring-page
   line art. Gap = Phase 0 prerequisite that doubles timeline.
2. Persistence: NEXT_PUBLIC_PERSISTENCE_ENABLED must be true (stable book IDs).
3. Cloudinary URLs must be public + stable (Pinterest fetches by URL).
4. Business entity: Etsy + Pinterest Business require registered seller ID.
5. Domain: Pinterest claim-website needs DNS access for crayonsparks.com.


================================================================================
PLATFORM 1 — ETSY
================================================================================

API: Etsy Open API v3
Base URL: https://openapi.etsy.com/v3/application/

AUTH MODEL: OAuth 2.0 with PKCE (refresh-token flow)
- "API key" / "keystring" is just the OAuth client_id, not a bearer token
- No raw API-key model — every call uses an access token

REQUIRED SCOPES:
- listings_w  (create / update listings)
- listings_r  (read listings)
- shops_r     (resolve shop_id)
- transactions_r (optional, for sales reporting)

WHERE TO OBTAIN:
- developers.etsy.com → register app under seller's Etsy account

APPROVAL TIMELINE (BLOCKER):
- Personal app keys: 1-2 days
- Commercial app review: 2-6 WEEKS (frequent first-pass rejections)
- ACTION: Apply on day 1, parallel to Phase 1 work

TOKEN STORAGE:
- Firestore document: users/{uid}/integrations/etsy
- refresh_token encrypted via Google Cloud KMS envelope encryption
- access_token (1h TTL) cached in memory only
- DO NOT use env vars — these are PER-USER tokens, not service account

RATE LIMITS: 10 req/sec/app, 10,000 req/day/app
- A 50-listing day at ~6 calls per listing = 300-600 calls (well within)
- Image-upload calls are slowest (~2-4s each)

LISTING CREATION FLOW (multi-step):
1. POST /shops/{shop_id}/listings           → create draft (title, desc, price,
                                                taxonomy_id, who_made,
                                                when_made, type=download)
2. POST /shops/{shop_id}/listings/{lid}/images  → upload 5-10 images
                                                   (multipart, one at a time)
3. POST /shops/{shop_id}/listings/{lid}/files   → upload PDF/ZIP digital files
4. PUT  /shops/{shop_id}/listings/{lid}     → state=active to publish
5. PUT  /shops/{shop_id}/listings/{lid}/inventory (optional, variants/SKUs)

REQUIRED FIELDS PEOPLE MISS:
- shipping_profile_id NOT required for digital (type=download)
- BUT return_policy_id MUST reference an existing policy
  → pre-create one default policy per seller
- who_made = "i_did" (with AI disclosure — see Compliance)
- when_made = "made_to_order" for AI-generated work
- taxonomy_id: coloring books → "Coloring Pages" (resolve once via
  /buyer-taxonomy/nodes, cache it)

IDEMPOTENCY: No native key.
- Store {book_id, platform=etsy} as unique key in publish_jobs collection
- State machine: pending → creating → uploading_images → uploading_files
                → activating → done/failed
- On retry: skip steps already marked complete
- Dup-publish detected by checking existing etsy_listing_id on book record

ASSET PREPARATION:
- Primary image: 2000x2000 square JPG/PNG, <20MB
  → letterbox 3:4 cover onto 1:1 canvas with brand-colored background
- Up to 10 images per listing. Recommended fill:
    1. Square cover
    2. Back cover
    3-7. Mockups (open-book, hand-coloring, framed)
    8. Sample interior page
    9. "What's included" infographic
    10. Bundle/contents grid
- Mockups reusable as-is. Square cover via Sharp/Canvas compositing
  (deterministic, free, fast — preferred over Gemini)
- Watermark only the sample interior page ("PREVIEW")
- Digital file: existing KDP-ready PDF (max 20MB, up to 5 files per listing)

LISTING COPY / SEO:
- Title: 140 chars. Reuse KDP title+subtitle, append SEO tail
  ("Printable Coloring Pages, Instant Download, Kids Activity")
- Description: PLAIN TEXT + line breaks (no HTML).
  Re-render KDP HTML through HTML-to-text. Append fixed footer
  (What's Included / How to Print / Terms of Use)
- Tags: 13 TAGS x 20 CHARS EACH (vs KDP's 7 of any length)
  → OpenAI generator: 7 KDP keywords → 13 short-tail Etsy tags
  → Coloring vs story books use different tag pools
- Materials: 13 slots, secondary tags ("PDF", "JPG", "PNG", "Digital Download")

PRICING:
- Coloring book (30 pages): $4.99-$7.99
- Story book (illustrated):  $6.99-$9.99
- Bundle (3 themed):         $12.99
- USD base; Etsy auto-converts. Sales tax auto-collected for US buyers


================================================================================
PLATFORM 2 — PINTEREST
================================================================================

API: Pinterest API v5
Base URL: https://api.pinterest.com/v5/

AUTH MODEL: OAuth 2.0 (standard or PKCE — both work)
- Standard flow fine for server-side (we're Next.js server routes)

REQUIRED SCOPES:
- boards:read
- boards:write
- pins:read
- pins:write
- user_accounts:read

WHERE TO OBTAIN:
- developers.pinterest.com (requires Pinterest Business account)
- Free conversion from personal account

APPROVAL TIMELINE (BLOCKER):
- Trial mode: works immediately for developer's own account (good for dev)
- Standard-access (production): 1-3 WEEKS review
- Requires working app demo + privacy policy URL

TOKEN STORAGE: Same KMS-encrypted Firestore pattern as Etsy
- Access token TTL: 30 days
- Refresh token rotates on use

RATE LIMITS: 1,000 calls/hour/user, 10 pins/sec
- 50-pin day = trivial
- Image-upload via URL is faster than direct upload

PIN CREATION FLOW (single-step):
- POST /pins  body: {
    board_id,
    media_source: { source_type: "image_url", url: "<cloudinary-url>" },
    title,
    description,
    link,         // CRITICAL — destination URL with UTM params
    alt_text
  }

REQUIRED FIELDS:
- board_id: pre-create boards. Plan: auto-create per book theme on first
  publish (e.g. "Animal Coloring Pages", "Bedtime Stories") via POST /boards
- link: destination URL (Etsy/Gumroad listing). Pins without link = ~0
  conversion. Add UTM params: ?utm_source=pinterest&utm_campaign={book_id}
- media_source: Pinterest fetches from Cloudinary URL (no multi-step upload)
- alt_text: required for accessibility, helps SEO

VIDEO PINS (Phase 2): media_source.source_type="video_id" + chunked upload
via /media endpoint. Out of scope for Phase 1.

IDEMPOTENCY: No native key.
- Track {book_id, platform=pinterest, variant_id} in publish_jobs
- 3-5 pin variants per book = multiple sub-jobs

ASSET PREPARATION:
- Pin image: 2:3 ratio (1000x1500 recommended)
- Square + 1:2.1 also accepted but suppressed in feed
- Front cover is 3:4 (close to 2:3 but not exact)
  → Recommend Gemini "pin variant" call: cover + mockup + overlay text
     ("FREE Printable Coloring Pages — 30 Designs!")
  → Fallback: crop 3:4 → 2:3 (loses ~10% top/bottom)
- 3-5 variants per book (Pinterest algorithm rewards multi-pin posting)
- Schedule 24-48h apart via scheduled_publish_time
- AVOID watermarks (Pinterest down-ranks heavily watermarked images)
- Small bottom-corner "crayonsparks.com" wordmark OK
- Best-performing mockup styles: "hand-coloring" > "open-book"

PIN COPY / SEO:
- Title: 100 chars. Hook + benefit
  ("30 Magical Unicorn Coloring Pages — Instant Printable PDF")
- Description: 500 chars. Keyword-dense, ends with hashtags
  → DIFFERENT shape than Etsy (Instagram-caption style, not product copy)
  → Need dedicated OpenAI prompt template
- Hashtags: 3-5 lowercase
  Coloring: #coloringpages #kidsactivities #printablecoloring
  Story:    #bedtimestory #kidsbooks #readaloud
- Story books: emphasize bedtime/calm/parents-of-young-kids language
- Coloring: emphasize activity/screen-free/road-trip language

PRICING: N/A (Pinterest is promotional, not a marketplace).
Track conversion via UTM on link field.

PLATFORM 3 — GUMROAD
================================================================================

API: Gumroad API v2
Base URL: https://api.gumroad.com/v2/

AUTH MODEL: OAuth 2.0 OR personal access token
- Recommend OAuth for multi-user
- Personal access token fine for single-creator MVP

REQUIRED SCOPES:
- edit_products
- view_sales (optional)

WHERE TO OBTAIN:
- gumroad.com/settings/advanced       (personal token, instant)
- gumroad.com/oauth/applications      (OAuth app, instant)

APPROVAL TIMELINE: NONE (no review process — this is why Gumroad is Phase 1)

TOKEN STORAGE: Same KMS-encrypted Firestore pattern

RATE LIMITS: Undocumented but lenient (~60 req/min observed). Not a concern.

PRODUCT CREATION FLOW:
*** BLOCKER TO VERIFY ***
The Gumroad public API is read-mostly. POST /products may NOT exist publicly
(removed 2023). Three options:

Option A (recommended Phase 1):
  - Pre-create one "template" product manually
  - Clone via dashboard CSV import or Zapier-style flow
  - Use PUT /products/:id to update + POST /products/:id/enable to publish
  - LIMITATION: full automation requires workaround

Option B:
  - Headless Stripe-on-top model
  - Custom checkout we build, Gumroad for fulfillment via email-delivery API
  - More work but fully automated

Option C:
  - Confirm whether Gumroad has restored POST /products publicly as of Apr 2026
  - If yes, simplest platform of the three

ACTION ITEM: Phase 0 spike (4h) to verify API status.
If A/C unavailable → DROP Gumroad from Phase 1, substitute Payhip or Sellfy
(both have full create-product APIs).

Assuming Option A works:
1. PUT  /products/{id}       → name, description, price, custom_permalink
2. POST /products/{id}/variants  → optional tier variants (basic/bundle)
3. File upload via dashboard (no API endpoint for new products)
   OR use content field with public URL
4. POST /products/{id}/enable  → publish

IDEMPOTENCY: Use custom_permalink = book_slug (deterministic URL = idempotent)

ASSET PREPARATION:
- Cover: 600x600 minimum, 1280x720 recommended. Square works best.
- Reuse Etsy 2000x2000 cover, downscaled
- Up to 10 preview images (gallery on product page)
- Reuse same image set as Etsy
- Digital file: KDP PDF + ZIP (max 16GB, no concerns)
- Watermark: standard preview on sample pages, none on cover

LISTING COPY / SEO:
- Name: 100 chars. Reuse KDP title + subtitle
- Description: MARKDOWN supported
  → Convert KDP HTML → Markdown
  → Add bullet feature list ("- 30 unique designs - Print-ready PDF
    - Includes coloring tips") — Gumroad UI emphasizes bullets
- Tags: up to 5 (subset of Etsy 13). Less SEO-critical
- Custom permalink: crayonsparks.gumroad.com/l/{book-slug}

PRICING:
- Coloring book: $5.00 (PWYW with $5 minimum is interesting top-of-funnel)
- Story book:    $8.00
- Bundle:        $15.00
- Gumroad fee: 10% + Stripe fees (vs Etsy 6.5%, but no listing fee)

COLORING BOOK vs STORY BOOK DIFFERENCES
================================================================================

| Aspect              | Coloring Book           | Story Book                |
|---------------------|-------------------------|---------------------------|
| Interior            | Line-art PNG, 8.5x11    | Full-color, varies        |
| Per-page text       | None / small caption    | Narrative paragraph(s)    |
| KDP emphasis        | printable, activity     | bedtime, read-aloud       |
| Etsy taxonomy       | Coloring Pages          | Children's Books          |
| Pinterest board     | Activity / Crafts       | Parenting / Bedtime       |
| Pricing tier        | Lower ($5-8)            | Higher ($7-12)            |
| Mockup styles       | Hand-coloring, crayons  | Open-book, child-on-lap   |
| Pin overlay copy    | "30 Designs!" "Print!"  | "Bedtime Story" "Ages 3-7"|

CODE DIVERGENCE: BookKind enum ('coloring' | 'story') threaded through the
publish pipeline, with platform-specific copy templates and mockup-selection
rules keyed off it.

GAP TO FLAG: If /api/generate doesn't yet produce story-book illustrated
pages with narrative text, that's PHASE 0 PREREQUISITE work that doubles
the timeline.

                                                 updated_at }
  publish_jobs/{jobId}                        — { uid, book_id, platforms[],
                                                 state, steps[], errors[],
                                                 created_at, updated_at }

JOB QUEUE (RECOMMENDED): Cloud Tasks + Cloud Run worker
- POST /api/publish enqueues task with 10-min deadline
- Worker route /api/publish/worker processes one platform at a time
- UI subscribes to Firestore for live status
- Etsy image-upload alone can take 30-60s/listing — sync route would timeout

UI: SINGLE "Publish" BUTTON → MODAL WITH PLATFORM CHECKBOXES
(rejected: per-platform tabs)
Justification: 90% of users publish to all three at once. Modal shows:
  1. Connected platforms (with "Connect" buttons for unconnected)
  2. Per-platform asset preview (collapsed by default)
  3. Per-platform copy preview with edit button
  4. "Publish" button → enqueues job
After publish: progress view with per-platform status rows.

FAILURE HANDLING:
- Each step idempotent and resumable
- On 5xx: exponential backoff, max 3 retries
- On 4xx: surface human-readable error with "Fix and Retry" affordance
- Partial publishes (Etsy ok, Pinterest fail) are visible — retry just failed


================================================================================
COMPLIANCE & POLICY RISKS
================================================================================

ETSY (HIGH RISK):
- AI DISCLOSURE MANDATORY (since 2024). Must disclose in description AND in
  production-method dropdown. Failure = shop suspension.
  → Auto-inject "This artwork was created using AI assistance and refined by
    the artist" into descriptions
  → Select appropriate production method
- Banned categories: pure-AI without human refinement banned in some.
  Our refine flow gives human-in-the-loop position (defensible).
- Trademark: Disney/Marvel character coloring books = auto-takedown
  → Manual checkbox v1: "I confirm no copyrighted characters"
  → OpenAI vision pre-check v2
- Age rules: digital downloads exempt from physical-product disclosures,
  but copy should target the parent (the buyer), not under-13s

PINTEREST (MEDIUM RISK):
- AI content allowed, auto-labeled with "AI-generated" badge
- Affiliate-style spammy posting → account suspension
  → Cap pin frequency at 5-10/day per account
- Community Guidelines: NO medical/health claims around children
  → Copy templates must not say "improves child development" etc.

GUMROAD (LOW RISK):
- AI content explicitly allowed. No disclosure required.
- Refund policy required (link to hosted policy on crayonsparks.com)

CROSS-PLATFORM:
- Privacy policy URL needed for Pinterest + Etsy reviews
- Terms of service URL likewise
- COPPA: don't collect data from buyers under 13 (current flow fine —
  buyers are parents)

RISKS & MITIGATIONS
================================================================================

- Etsy app review rejects on AI grounds
  → Lead with human-in-the-loop refine flow; fallback to curated subset
- Gumroad public API doesn't support product creation
  → 4h Phase 0 spike confirms; pivot to Payhip

  → Unique key {book_id, platform} on publications collection

  → Content-moderation gate before publish, manual v1 → vision v2
